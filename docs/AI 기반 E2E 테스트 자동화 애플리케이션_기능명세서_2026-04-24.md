## 1. 프로젝트 개요
---
### 카테고리
> 생산성 / 업무

### 한 줄 정의
AI 기반 E2E 테스트 자동화 애플리케이션

### 배경
소프트웨어 개발 과정에서 E2E 테스트는 제품 품질을 보장하는 핵심 단계이지만, 수동 시나리오 작성에 소모되는 시간과 비용, 그리고 복잡한 자동화 환경 구축의 어려움으로 인해 많은 팀들이 비효율적인 프로세스를 겪고 있습니다. 이 제품은 이러한 문제를 해결하고, 개발 및 QA팀이 더 빠르고 정확하게 제품을 검증할 수 있도록 지원하기 위해 기획되었습니다.

### 사용자 문제
수동 테스트 시나리오 작성에 소모되는 엄청난 시간과 비용, E2E 테스트 자동화 환경 구축의 어려움, 그리고 산재된 테스트 결과에 대한 통합적인 추적 및 분석의 비효율성으로 인해 제품 출시 지연과 잠재적 버그 발생 위험이 증가합니다.

### 해결 방식
AI 에이전트가 개발 서버 URL을 분석하여 E2E 테스트 시나리오를 자동으로 추출하고, 추출된 시나리오를 기반으로 실제 브라우저에서 자동화된 테스트를 실행합니다. 모든 테스트 결과는 시각화된 대시보드를 통해 실시간으로 제공되어, 사용자들은 복잡한 설정 없이도 효율적인 테스트 자동화 및 관리를 경험할 수 있습니다.

### 차별점
기존의 E2E 테스트 솔루션들이 대부분 수동으로 시나리오를 작성하거나 코드를 통한 복잡한 설정이 필요한 반면, 본 제품은 AI 에이전트가 개발 서버 URL만으로 웹사이트를 자동으로 탐색하고 테스트 시나리오를 추출한다는 점에서 독보적인 차별점을 가집니다. 또한, 심플하면서도 직관적인 인터페이스와 데이터 시각화에 집중한 대시보드는 사용자 경험을 한층 더 향상시킬 것입니다.

### 타겟 사용자
프로덕트 매니저(PM), 개발자, QA 엔지니어. 특히, 반복적인 수동 테스트 작업에 지쳐있거나, 자동화된 테스트 환경 구축에 어려움을 겪는 중소기업 또는 스타트업의 IT팀이 주요 타겟입니다. 이들은 신속하고 정확한 테스트를 통해 제품 품질을 높이고 개발 주기를 단축하고자 하는 니즈를 가지고 있습니다.

### 사용 시나리오
QA 엔지니어 김대리(3년차)는 새로 개발된 서비스의 기능들을 E2E 테스트해야 합니다. 김대리는 이 애플리케이션에 접속하여 개발 서버 URL을 입력합니다. 잠시 후 AI 에이전트가 웹사이트를 분석하여 로그인, 게시글 작성, 결제 등 주요 사용자 흐름에 대한 테스트 시나리오를 자동으로 추출하고 엑셀 파일로 제공합니다. 김대리는 추출된 시나리오를 검토하고 필요한 부분을 수정하거나 추가한 후, '테스트 실행' 버튼을 클릭합니다. 애플리케이션은 시나리오에 따라 브라우저에서 자동으로 테스트를 수행하고, 완료 후 대시보드에 접속하여 각 시나리오별 성공/실패 여부, 성능 데이터, 오류 스크린샷 등을 한눈에 확인합니다. 김대리는 이 보고서를 팀에 공유하여 빠르게 문제점을 파악하고 개선할 수 있게 됩니다.

### 제품 목표
사용자(PM, 개발자, QA)가 개발 서버 URL만으로 E2E 테스트 시나리오를 자동으로 추출하고, 브라우저 기반으로 테스트를 수행하며, 결과를 시각화된 대시보드를 통해 쉽게 확인할 수 있도록 하여 테스트 프로세스를 혁신적으로 자동화하고 효율성을 극대화합니다.

### 핵심 KPI
월별 자동화 테스트 실행 횟수 (예: 월 20% 증가), 신규 시나리오 추출 성공률 (예: 80% 이상), 테스트 사이클 시간 단축률 (예: 30% 단축), 사용자 재방문율 및 월간 활성 사용자(MAU) 증가, 사용자 피드백을 통한 만족도 점수 (예: 5점 만점에 4점 이상).

### 리스크/이슈
AI 시나리오 추출의 정확도 및 커버리지 문제가 발생할 수 있으며, 특히 복잡하거나 동적인 웹사이트의 경우 AI 모델이 모든 엣지 케이스를 완벽하게 이해하지 못할 가능성이 있습니다. 또한, 대상 웹사이트의 기술 스택 및 구조에 따라 브라우저 자동화 테스트의 안정성이 저해될 수 있습니다. Anthropic Claude Opus 4.7 모델 사용에 따른 높은 API 호출 비용과 경쟁사 유사 솔루션 등장에 대한 시장 선점 경쟁도 중요한 위험 요소입니다. 개발 서버 URL 접근 및 테스트 데이터 처리 과정에서의 보안 취약점도 철저히 관리해야 합니다.

## 1. AI 기반 E2E 테스트 시나리오 자동 추출
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 개발 서버 URL을 입력하면, AI 에이전트가 해당 웹사이트를 탐색하여 주요 사용자 흐름에 대한 E2E 테스트 시나리오를 자동으로 추출하고, 이를 엑셀 파일로 다운로드할 수 있도록 제공합니다.

**수용 기준:**
```
1. ○ 사용자는 개발 서버 URL을 입력할 수 있어야 한다.
2. ○ AI 에이전트는 입력된 URL을 기반으로 웹사이트를 분석하여 테스트 시나리오 목록을 생성해야 한다.

4. ○ 사용자는 생성된 시나리오 목록을 엑셀(XLSX) 파일 형식으로 다운로드할 수 있어야 한다.
```

## 2. 자동화된 브라우저 기반 E2E 테스트 실행
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

추출되거나 사용자가 직접 수정한 시나리오를 바탕으로 실제 브라우저 환경에서 자동화된 E2E 테스트를 수행하며, 테스트 진행 상황을 실시간으로 확인할 수 있도록 합니다.

**수용 기준:**
```
1. ○ 사용자는 선택한 시나리오 또는 전체 시나리오에 대해 테스트 실행을 시작할 수 있어야 한다.
2. ○ 테스트는 실제 브라우저(예: Chrome, Firefox) 환경에서 자동으로 수행되어야 한다.
3. ○ 테스트 실행 중 각 단계의 성공/실패 여부를 실시간으로 표시해야 한다.
4. ○ 테스트 실패 시 해당 지점의 스크린샷과 오류 로그를 기록해야 한다.
```


### 2.1 브라우저 자동화 테스트 실행기
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: Developer, Quality Assurance Engineer | **기기**: Web

사용자가 선택한 테스트 시나리오를 실제 브라우저 환경에서 자동으로 실행하는 기능으로, Chrome과 Firefox 등 주요 브라우저를 지원합니다.

## 6. 기술 스택 및 인프라 구현

**수용 기준:**
```
1. ○ 프론트엔드 및 백엔드 개발은 Next.js를 기반으로 이루어져야 한다.
2. ○ UI/UX는 Tailwind CSS를 활용하여 일관된 디자인 가이드라인을 따라야 한다.
3. ○ 데이터베이스 및 인증, API 게이트웨이는 Supabase를 통해 구축되어야 한다.
4. ○ AI 기반 시나리오 추출 기능은 Anthropic Claude Opus 4.7 API를 사용하여 구현되어야 한다.
```
